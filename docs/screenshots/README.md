# Screenshot Checklist

Screenshots are not committed yet. Capture real screenshots only after the app is connected to Supabase demo data and the relevant flows have been verified. Do not add mock or fabricated screenshots.

Expected files:

- `landing.png` - landing page with public-safe stats.
- `new-report.png` - protected issue submission form with map picker.
- `issue-list.png` - public issue list with filters and pagination.
- `issue-detail.png` - public issue detail with image/timeline/map preview.
- `public-map.png` - public Leaflet map with marker popup and OpenStreetMap attribution.
- `admin-dashboard.png` - admin dashboard with moderation queue.
- `admin-issue-detail.png` - admin moderation page with status dialog, notes, and notification log.
- `analytics.png` - admin analytics cards and charts.
- `discord-alert.png` - real Discord alert generated by a high or critical issue.

Recommended capture notes:

- Use seeded/demo data with a mix of statuses, categories, and urgency levels.
- Capture desktop screenshots first, then optionally add mobile screenshots later.
- Avoid showing private secrets, webhook URLs, service-role keys, or real personal data.
- If a Discord webhook is not configured, skip `discord-alert.png` until a real alert can be captured.
